Docker安装

问题：sudo apt install docker.io docker-compose-plugin -y 报 docker-compose-plugin has no installation candidate

原因：docker-compose-plugin 是 Docker 官方源提供的包，Kali 默认源不包含它。

问题：docker run hello-world 报错 context deadline exceeded

原因：Kali 虚拟机网络能通，但无法访问 Docker Hub 官方仓库，拉取镜像超时。

问题：pip install docker-compose 报 externally-managed-environment

原因：新版 Kali Python 启用了 PEP 668 保护，不允许 pip 直接修改系统环境。

问题：Kali 中 Docker 安装后无法拉取镜像，报错 context deadline exceeded，但虚拟机可以正常访问百度等网页。

原因：校园网防火墙屏蔽了 Docker Hub 境外仓库的 HTTPS 流量，普通网页不受影响，但 Docker 镜像被拦截。

get和post

在客户机和服务器之间进行请求-响应时，两种最常被用到的方法是：GET 和 POST。

  • GET - 从指定的资源请求数据。
  • POST - 向指定的资源提交要被处理的数据。

GET 提交参数一般显示在 URL 上，POST 通过表单提交不会显示在 URL 上，POST 更具隐蔽性

robots协议

robots协议也称爬虫协议、爬虫规则等,是指网站可建立一个robots.txt文件来告诉搜索引擎哪些页面可以抓取,哪些页面不能抓取,而搜索引擎则通过读取robots.txt文件来识别这个页面是否允许被抓取。

Linux命令拼接

Linux 常用基础命令
一、目录操作

  1. ：查看目录内容

    • ls -l 详细信息
    • ls -a 显示隐藏文件

  2. pwd：查看当前所在路径

  3. ：切换目录

    • cd ~ 回到家目录
    • cd .. 返回上一级
    • cd / 进入根目录

  4. mkdir 目录名：创建文件夹

  5. rmdir 目录名：删除空目录

二、文件操作

  1. touch 文件名：创建空文件

  2. ：删除文件

    • rm -r 目录 递归删除文件夹
    • rm -rf 强制删除

  3. ：复制文件 / 目录

    • cp -r 复制文件夹

  4. mv 源 目标：移动 / 重命名

  5. ln：创建软硬链接

三、文件查看与编辑

  1. cat 文件名：一次性查看全部内容

  2. more/less 文件名：分页查看长文件

  3. head -n 数字 文件名：查看前几行

  4. ：查看末尾几行

    • tail -f 实时监控日志

  5. 编辑：vi / vim 文件名

四、搜索与查找
  1. find 路径 -name "关键词"：查找文件
  2. grep 关键词 文件名：文件内搜索内容
  3. which 命令：查看命令所在路径
五、系统信息
  1. uname -a：系统内核信息
  2. df -h：查看磁盘占用
  3. du -sh：查看文件 / 文件夹大小
  4. free -h：查看内存
  5. top：实时查看进程、资源占用
六、权限管理
  1. chmod 权限 文件：修改文件权限
  2. chown 用户:组 文件：修改所有者
  3. whoami：查看当前用户
  4. su / sudo：切换 root / 临时提权
七、压缩解压

  1. 打包

    • 打包：tar -cvf 压缩包.tar 目标
    • 解包：tar -xvf 压缩包.tar

  2. 常用压缩

    • 压缩：tar -zcvf xxx.tar.gz 文件
    • 解压：tar -zxvf xxx.tar.gz
八、网络命令
  1. ip addr / ifconfig：查看网卡 IP
  2. ping 域名/IP：测试网络连通
  3. curl 网址：访问网页 / 接口
  4. netstat：查看端口占用
九、进程管理
  1. ps aux：查看所有进程
  2. kill 进程ID：结束进程
  3. kill -9 进程ID：强制杀死进程
十、其他实用
  1. clear：清空终端
  2. history：查看历史执行命令
  3. date：查看系统时间
  4. halt / reboot：关机、重启

Vim备份文件后缀

在使用Vim编辑器时，Vim会生成一些临时文件和备份文件，以确保在编辑过程中数据不会丢失。这些文件的后缀各不相同，具体如下：

交换文件（Swap Files）

交换文件用于在编辑过程中保存文件的临时副本，以防止数据丢失。交换文件的后缀通常为*.swp*。如果Vim异常退出，交换文件不会被删除，可以通过它恢复未保存的内容。例如：

filename.swp

如果多次异常退出，交换文件的后缀会依次变为*.swo*、.swn等。

备份文件（Backup Files）

备份文件用于保存文件的旧版本，以便在需要时恢复。备份文件的后缀通常为~。例如：

filename~

可以通过在Vim配置文件中设置set backup来启用备份文件。

撤销文件（Undo Files）

撤销文件用于保存文件的修改历史，以便在需要时撤销更改。撤销文件的后缀通常为*.un~*。例如：

filename.un~

可以通过在Vim配置文件中设置set undofile来启用撤销文件。
