二.下载git

Git 是本地环境与 GitHub 之间的标准通信工具与版本控制核心。Hexo 仅负责静态网页生成，不具备网络传输能力；必须依靠 Git 完成文件推送、版本管理与部署操作，才能使博客正式发布到 GitHub Pages 并对外访问。

下载地址：

https://git-scm.com/downloads](https://git-scm.com/downloads

三.生成SSH Keys

SSH Key 是一种基于 非对称加密算法 的安全认证机制，由一对密钥构成：

  • 私钥（id_rsa）：保存在本地，用于身份签名
  • 公钥（id_rsa.pub）：上传到 GitHub，用于验证身份

生成 SSH Key 是为了在本地与 GitHub 之间建立基于非对称加密的安全通信通道。

认证成功后，Git 可通过加密通道实现免密提交、推送与部署，保障传输安全性与效率。

四.本地搭建博客

创建一个名为 Blog 的文件，在里面启用 Git Bash Here

（Git Bash 是命令行终端，提供完整的 Git、系统与 SSH 功能，适合开发与部署；Git GUI 是图形化工具，仅提供基础可视化操作，功能有限。Hexo + GitHub 博客搭建必须使用 Git Bash，因为 SSH 配置、密钥生成、部署命令均依赖命令行环境。）

初始化hexo

初始化时出现报错

Git SSL证书错误：unable to get local issuer certificate 是由于Git客户端无法验证GitHub的SSL证书导致的，通常是因为系统没有安装根证书、网络环境有代理/防火墙拦截，或者Git的SSL验证配置开启但证书链不完整。

权限不足错误：EPERM: operation not permitted, mkdir 'E:\' 是因为当前在Git Bash中没有E盘根目录的写入权限，导致Hexo在Git克隆失败后尝试直接在E盘根目录创建文件夹时被系统拒绝。

但是解决后仍然出现问题

五.上传到github

修改 _config.yml 的 deploy 配置，向 Hexo 声明部署方式与目标仓库，建立本地与 GitHub 之间的部署规则。type: git 确定使用 Git 协议传输，repository 指定目标仓库，二者配合实现 Hexo 静态网站自动化部署到 GitHub Pages。

deploy:

	 type: git

​ repository: github地址(要先在自己的github上创建一个仓库)

	 branch: main

出现错误

Git推送时的SSL证书校验问题

Git 推送时出现 SSL 证书校验错误，是因为采用 HTTPS 协议传输。

采用解决方法是通过配置 SSH Key，并将仓库地址切换为 SSH 协议，可以绕过 HTTPS 的 SSL 证书验证流程，利用非对称加密实现本地与 GitHub 的信任连接，从根本上解决 SSL 校验失败问题。
